# 股票详情页设计

日期：2026-07-16  
状态：待实现

## 背景与目标

刻度 Desk 已有行情 bars、盘中 quote、标的 meta、板块成员等能力，但缺少统一的「单标的深看」体验。目标：

1. 提供股票详情：**实时行情、分时/K 线、基本信息、行业/概念、资金面、技术面**。
2. 从 **实盘监控持仓、自选、顶部搜索** 都能进入。
3. **默认抽屉速览，可展开全页**；信息量大时全页为主阅读面。
4. 数据 **库表优先**；缺口用 **AkShare / xtdata 现拉**，并可落库，不阻塞第一版上线。

## 已确认决策

| 项 | 选择 |
|----|------|
| 入口 | 持仓 + 自选 + 顶部搜索跳转 |
| MVP 模块 | 行情头、图表、基本信息、行业/概念、资金面、技术面（全部） |
| 数据策略 | 库优先；缺口 live 拉/算；可选落库 |
| 呈现 | 抽屉速览 + 「展开」进入全页 |
| 图表周期 | 分时 + 日 / 周 / 月 K |
| 仓位上下文 | 从持仓进入时抽屉带摘要；全页可收起；自选/搜索不带 |
| 技术路线 | 前端拼装 + 薄聚合接口（复用已有 API，新增资金/技术） |

## 非目标（第一版）

- 不做 WebSocket 实时推送（可后续加轮询）。
- 不做分钟级多周期 K（1/5/15/60）；仅分时 + 日/周/月。
- 不做单一「详情大接口」一次返回全量。
- 不把详情做成纯 mock 演示页。
- 行业/概念若尚无板块详情页，第一版只展示标签，不强制新路由。

## 架构

```
入口（持仓 / 自选 / 搜索）
        │
        ├─ 抽屉 overlay（可带 position state）
        │       └─ 「展开」→
        └─ 全页 /stock/:symbol
                │
                └─ StockDetailView（共用）
                        │
        ┌───────────────┼───────────────┐
        │ 已有 API      │ 新增 API      │
        │ quote         │ capital-flow  │
        │ bars          │ technicals    │
        │ meta / boards │               │
        └───────┬───────┴───────┬───────┘
                │               │
           Postgres 优先    缺口 live（AkShare / xtdata）
```

### 前端

- 共享组件：`StockDetailView`。
- 全页路由：`/stock/:symbol`（壳层导航保留）。
- 抽屉：监控/自选触发；顶栏「展开全页」+ 关闭。
- 搜索：选中标的直接进全页（不经抽屉）。
- 仓位摘要：仅 location state / props 传入，不新增仓位 API。

### 后端

- **复用**：`/api/market/...` 下 bars（日/分）、intraday quote、security_meta、boards / board_members。
- **新增**：
  - `GET /api/market/stock/{symbol}/capital-flow`
  - `GET /api/market/stock/{symbol}/technicals`
- 周/月 K：优先由日 K 聚合；历史不足时补拉日线再聚合。
- 响应可带 `source: "db" | "live"`，便于排查。

## 页面结构

### 全页（自上而下）

1. **行情头**：代码、名称、现价、涨跌幅、量/额、刷新时间。
2. **仓位摘要（可选）**：成本、浮动盈亏、仓位占比；可收起。仅从持仓带入时出现。
3. **图表区**：Tab = 分时 | 日 | 周 | 月。
4. **下方模块**（桌面多列，窄屏纵向）：基本信息、行业/概念、资金面、技术面。

### 抽屉

- 同序纵向压缩：头 +（可选仓位）+ 图表 + 折叠面板装后四块。
- 单模块失败不影响其它模块。

## 数据流与错误处理

- 打开后并行请求：quote、meta、boards、bars（当前周期）、capital-flow、technicals。
- 切换图表周期只重拉/重算图表相关数据，其它模块不重置。
- live 失败或超时：该模块空态/错误条；整页不崩。建议单接口超时约 8～15s。
- 未知 symbol：全页「未找到标的」+ 回监控。
- DB 不可达：沿用壳层健康提示；能出的模块照常。
- 手动刷新按钮重拉并行请求；第一版无强制轮询。

## API 契约（草案）

### `GET /api/market/stock/{symbol}/capital-flow`

- 成功：`available: true`，含主力/分区净流入（以所选数据源能提供的字段为准）、可选近 N 日序列、`source`、`as_of`。
- 无数据或 live 失败：HTTP **200** + `available: false` + 可选 `error` 文案；**不抛未处理 500 打断整页**。非法 symbol：**404**。

### `GET /api/market/stock/{symbol}/technicals`

- 第一版固定指标：`MA5` / `MA10` / `MA20`、`MACD`（dif/dea/hist）、`RSI14`。
- 基于日 K（不足则补拉）计算；返回最新值 + 与日 K 对齐的序列（供叠加）；`source`、`as_of`。
- 计算失败时同样 **200** + `available: false`。

字段命名与现有 market API 一致（snake_case JSON）；表结构若需新建，在实现计划中给出 DDL。

## 测试要点

- 持仓/自选打开抽屉；展开后 URL 与 symbol 一致。
- 搜索直达全页。
- 仓位摘要：持仓进入可见可收起；自选/搜索不出现。
- 分时/日/周/月切换；图表失败不影响其它模块。
- capital-flow / technicals：有库走库、无库 live；live 失败可解析空态。
- 非法 symbol → 404。
- 手工：一只库表完整票、一只缺资金/技术票各走一遍抽屉+全页。

## 实现顺序（建议）

1. 路由 + `StockDetailView` 骨架 + 抽屉/展开。
2. 接入已有 quote / bars / meta / boards。
3. 新增 capital-flow、technicals 与空态。
4. 持仓/自选/搜索入口接线 + 仓位摘要。
5. 验收与补测。

## 后续可演进（非本 spec）

- 盘中轮询或 WebSocket。
- 分钟多周期 K。
- 将 live 结果固化为定时同步任务。
- 概念/行业跳转板块详情页。
