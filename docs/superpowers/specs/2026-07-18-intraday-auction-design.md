# 分时图开盘集合竞价段设计

日期：2026-07-18  
状态：待实现  
前置：`2026-07-16-stock-detail-design.md`（详情页分时已落地）

## 背景与目标

当前分时图会话轴从 **09:30** 起算，`toAshareSessionIndex` 会丢弃开盘前数据；前端请求也从 `09:30` 起拉分钟线。目标：在分时图左侧展示 **开盘集合竞价及开盘前空档（09:15–09:30）**，与连续竞价段视觉区分，数据优先走 **QMT / xtdata**。

## 已确认决策

| 项 | 选择 |
|----|------|
| 时段 | **09:15–09:30**（含 09:25–09:30 空档） |
| 呈现 | **同一根分时轴**；左侧竞价区浅底色 + **09:30 竖线** 与连续竞价分隔 |
| 价/量 | 竞价段与连续竞价段均绘制价格与成交量 |
| 摘要 | 顶栏均价 / OHLC **仅统计 ≥09:30** 的连续竞价段，避免竞价量扭曲均价 |
| 十字光标 | 正常显示 `09:xx`（含竞价分钟） |
| 数据源 | QMT `1m` 优先；不足再用 tick 聚合成 1 分钟 |
| 收盘竞价 | **不做**（14:57–15:00 本迭代不单列） |

## 非目标

- 不做收盘集合竞价独立分区。
- 不做竞价小图 + 主图并排双图表。
- 不改变日 / 周 / 月 K 行为。
- 不强制全市场落库竞价分钟；详情分时仍按现有「库优先 + live 补洞」路径，仅把时间窗前移到 09:15。

## 坐标与刻度

连续交易轴在现有「跳过午休」模型上**前插 15 分钟**：

| 北京时间 | 会话序号（示意） |
|----------|------------------|
| 09:15 | 0 |
| 09:25 | 10 |
| 09:30 | 15 |
| 11:30 / 13:00 | 15 + 120 = 135 |
| 15:00 | 15 + 240 = 255 |

- 刻度至少展示：`09:15`、`09:30`、`11:30/13:00`、`15:00`。
- `INTRADAY_TIME_BASE` 伪时间基数可保留；仅会话序号定义变更。
- 竞价区视觉：图表背景在序号 `[0, 15)` 使用浅底色（或等价叠加），并在序号 `15` 画竖线。

实现时集中改 `apps/web/src/stock/format.ts` 与 `StockChart.tsx`；配套单测更新 `format.test.ts`。

## 数据流

```
StockDetailView loadBars(intraday)
  from: 当日 09:15 Asia/Shanghai
  to:   当日 15:00（或现有截断逻辑）
        │
        ▼
GET /api/market/bars?...（或现有分钟路径）
        │
        ├─ 库 bars_minute（若已有 09:15+）
        └─ QMT get_minute_bars(1m) → 缺口则 tick→1m 聚合
        │
        ▼
toChartBars / toIntradayChartTime（含 09:15–09:29）
        │
        ▼
StockChart：占位轴 0…255 + 竞价底色 + 价/量/MACD
```

### 后端 / QMT

- `get_minute_bars` 查询窗允许 `09:15` 起；若 xtdata `1m` 不含竞价，在 `qmt_md`（或紧邻适配层）增加 **tick 拉取并按分钟 OHLCV 聚合**，仅用于补 09:15–09:29（及必要时 09:25–09:30 空档的跳空处理）。
- 空档分钟：无成交则可不插点，或用上一有效竞价价平铺——**优先不插假量**；价格线允许在竞价末与 09:30 首根之间自然断档/衔接，以实际 QMT 返回为准。
- MockQmt 需能 seed 09:15 起分钟，便于单测。

### 前端摘要

- `summarizeIntradayBars`：过滤会话序号 `< 15`（或墙钟 `< 09:30`）后再算均价/OHLC。
- 成交量副图：竞价段柱正常显示。

## 错误处理

| 情况 | 行为 |
|------|------|
| QMT 无竞价分钟/tick | 分时仍从首根有效连续竞价画起；竞价区可为空但轴与底色仍在 |
| 仅有部分竞价分钟 | 有则画，不因缺 09:15 整段失败 |
| 非交易日 / 无数据 | 与现网分时空态一致 |

## 测试

- `format.ts`：`09:15→0`、`09:30→15`、刻度文案、竞价点不丢弃。
- `summarizeIntradayBars`：含竞价量时均价仍只看 ≥09:30。
- 有 Mock/QMT fixture 时：`get_minute_bars` 或聚合路径返回 09:15–09:30 样本。
- 图表：手动确认左侧底色与 09:30 竖线（实现后验收）。

## 验收标准

1. 分时请求时间窗包含当日 **09:15**。
2. 有竞价数据时，左侧可见 **09:15–09:30** 价/量，并有底色 + 09:30 分隔。
3. 顶栏均价/OHLC **不含** 竞价段成交。
4. 无竞价数据时，连续竞价分时仍可用，不报错。
