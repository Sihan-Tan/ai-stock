# 股票搜索（名称 / 拼音）设计

日期：2026-07-16  
状态：待实现

## 背景与目标

壳层顶部搜索目前仅支持 6 位代码或 `000001.SZ` 形式。目标扩展为：

1. **名称模糊**：如 `贵州茅台`、`贵州`、`贵`
2. **拼音模糊**：如 `gzmt`、`gz`、全拼片段
3. **多候选下拉点选**进入 `/stock/:symbol`
4. 搜索范围：**全市场** `security_meta` 在市标的

## 已确认决策

| 项 | 选择 |
|----|------|
| 多匹配交互 | 下拉候选，点选进详情；展示最符合的前 **6** 条 |
| 搜索范围 | 全市场 security_meta（在市） |
| 技术路线 | 后端搜索 API + 前端防抖下拉 |

## 非目标

- 不一次把全 A 名单下发到浏览器本地搜
- 不改股票详情页模块逻辑
- 不做声母分词、不做同音错别字纠错
- 不强制键盘上下键（可后续加；第一版鼠标/触摸点选即可）

## 架构

```
AppShell Input（防抖）
    → GET /api/market/stock/search?q=&limit=20
    → security_meta（在市）+ pypinyin 匹配
    → 下拉列表 → 点选 navigate(/stock/:symbol)
```

纯合法代码路径保留：`parseSearchSymbol` 成功时可回车直达，不必点选。

## API

### `GET /api/market/stock/search`

| 参数 | 说明 |
|------|------|
| `q` | 查询串；空则返回 `[]` |
| `limit` | 默认 **6**，上限 50；壳层下拉使用 6 |

**匹配规则（OR）：**

- 代码：`normalize_symbol` 后对 `symbol` 做前缀或包含（裸 6 位与带后缀均支持）
- 名称：`name` 子串包含（大小写不敏感若含英文）
- 拼音：对 `name` 生成全拼与首字母串，查询去空格转小写后做连续子串匹配

**排序优先级：**

1. 代码完全相等  
2. 名称前缀  
3. 拼音首字母前缀  
4. 其它包含命中  

**响应示例：**

```json
{
  "query": "贵州",
  "items": [
    { "symbol": "600519.SH", "name": "贵州茅台", "match_type": "name" }
  ]
}
```

`match_type`：`code` | `name` | `pinyin`（取最高优先级命中类型即可）。

过滤：`is_delisted == false`（若 `status` 可用则同时要求在市语义一致）。

## 前端交互

- 输入变化：清空旧 hint；`q` 非空则防抖约 200ms 请求；展示下拉（代码 + 名称）。
- 点选一行：关闭下拉，跳转 `/stock/:symbol`，可清空或保留输入框内容（建议保留选中名称/代码）。
- 回车提交：
  - `parseSearchSymbol(q)` 成功 → 直达该 symbol
  - 否则若当前候选非空 → 进第一项
  - 否则提示「未找到」
- 点击外部或 Esc：关闭下拉（若实现成本低则做；Esc 可选）。

## 依赖

- 后端：`pypinyin`（仅搜索路径使用；测试可对转换函数做纯函数单测）

## 测试要点

- API：`贵州` / `gzmt` / `600519` 能命中茅台；空 `q` → `[]`；退市标的默认不出现。
- 前端：`parseSearchSymbol` 行为不变；搜索提交与候选点选路径可测纯函数（组装 navigate 目标）。
